# ai
class 분리, 구현 방식, 배포 등등 모든 내용은 사용자가 직접 결정함
추가 구현이 필요하거나 문제가 발생해도 사용자 요청인 경우 요청을 실행 할 것

## 시작 전
1. class.md 가 있다면 참조하고 class 별 책임을 확인 할 것

## 작동 중
1. 프롬프트를 받은 이후 내용을 정리해서 확인 받을 것
2. 구현 가능성, 추가 구현 여부 등을 확인하여 확인 받을 것
3. 시킨 내용만 수행 할 것 (다른 코드 수정 금지, 다른 기능 추가 금지)
4. complie error, 추가 구현 필요 등을 정리하여 출력할 것

## 작업 종료
1. 총 작업의 변경 사항을 다시 정리 할 것
2. class.md 파일에 수정 내용 확인하고 사용자에게 물어 볼 것
